# 弓道射法八節動作解析アプリーインストール手順  

## １）ZIPファイルをダウンロードして解凍する  

	１．zipファイルを"右クリック＞全て展開"で解凍する  
	２．展開先フォルダーに”c:\Users\ユーザー名\”を指定する  
	３．展開後のトップフォルダーを"YOLO"にリネームする
		
## ２）pythonをMicrosoft Storeからインストールする

	 ・インストールの有無、インストール済pythonのバージョンはコンソールから確認  
        > python -V  
    （注）Visual Studio(C:\ProgramFile(x86)\Microsoft VisualStudio)がないときは、  
          Visual Studio Community206のインストールが必要。  
          [ダウンロードサイト](https://visualstudio.microsoft.com/ja/vs/community/)  
          （インストール時に、「Python開発」にチェックを入れる）
		
## ３）ターミナル（管理者）、またはPowerShell（管理者）でコンソールを開く（タスクバーのスタート右クリックメニュー）

###	１.アプリ実行フォルダーに移動して、セットアッププロファイルを作成する。  
	   （コンソールオープン時に自動的に実行されるユーザープロファイルも作成可  
	   
	    >cd c:\Users\ユーザー名\YOLO   
	    >python ./src/setup.py $profile  
		
###	２．セットアッププロファイルを実行する。 

	    >./SetupKyudo.ps1    
	
	　		① ユーザー許可のプロファイル実行ポリシーを恒常的に設定する。  
				> Set-ExecutionPolicy RemoteSigned  
	　		② カレントのフォルダー位置をアプリの実行フォルダーに設定する  
	　		③ 動画ファイルを保存するデフォルトのフォルダーを設定する  
	　		④ 必要に応じて、Pythonの仮想環境を作成する  
	　		⑤ アプリ実行に必要なパッケージをインストールする  
	  		（注）アンチウィルスソフトのリアルタイムスキャンを停止しておく  
				> python -m pip install -r  requirments.txt  
		
# 実行手順  

## １）ターミナル、またはPowerShellでコンソールを開く。 

	（※）ユーザープロファイル未作成のとき、StartKyudo.ps1を実行する  
	
	> ./StartKyudo.ps1  
			
## ２）helpメニューに従ってコンソールからコマンドを入力する。  	

# 実行例

## １）仮想環境をアクティブにする（セットアップで仮想環境を作成したとき）

		>actvenv
		-プロファイルの実行でアクティブになっている場合は不要

## ２）既定の実行環境変数を表示する

		>model

## ３）学習時のハイパーパラメータを変更する

		>model -hp (96, 64, 300, 0.1)  
		
		-ミニバッチサイズ=64,エポック数=300,学習率の低減率=0.1に変更する

## ４）動画ファイルを生再生する

		>yoloap -raw  
		
		-ファイル選択ダイアログで再生ファイルを指定する
		-キーボード操作により各種再生モードの切り替え可能

## ５）二つの動画ファイルを重ねて再生する

		>yoloap -multi '100,200'
		
		-ファイル選択ダイアログで再生ファイルを二つ指定する
		-再生開始フレームを第一、第二ファイルそれぞれ100,200に指定する

		>yoloap -comp 'yoshimoto_2m.0,yoshimoto_2m.9' -at '100,200'

		-登録済ケース名を指定して、二つの動画を重ねて再生する  
		 ※）ケース名の登録は10)を参照
		 -再生開始フレームを第一、第二ファイルそれぞれ100,200に指定する
		-指定ケース名は一つだけでも可。この場合は、骨格解析の描画が可能。
		
## ６）動画ファイルをクリッピングして別ファイル（ファイル名先頭に'_'付加）に保存する

		>yoloap -clip  
		
		-ファイル選択ダイアログでクリッピング対象ファイルを指定する
		-マウスドラッグによる矩形で切り取り範囲を指定
		-二番目以降の矩形でモザイク処理をする範囲を指定可能
		-'-rotate'オプションを追加指定することで、画像を９０度時計回りに回転して再生可能
		-ファイル出力開始／停止は'w'キーで操作する

## ７）射形動画をプログラムロジックで射法八節姿勢解析し、動画に表示・再生する

		>yoloap -man  
		
		-ファイル選択ダイアログで解析対象ファイルを指定する
		-デフォルトの骨格解析はYOLOv8s(-v8 s)
		-デフォルトの八節解析パラメータレベルは2(-level 2)
		-デフォルトのデータサンプリングは１フレーム毎、比較フレーム間隔は７フレーム(-sample 1.7)
		-八節解析の開始、節開始/完了の強制指定はキー操作('0', '1'-'9'/スペース)で行う

## ８）射形動画をプログラムロジックで射法八節姿勢解析し、再生結果を別ファイルに保存、解析データをCSVファイルに出力する

		>yoloap -case yoshimoto_2m.0 -v8 m -level 0 -sample 1.10  
		
		-骨格解析はYOLOv8m,八節解析パラメータレベルは0,データサンプリングは1.10を指定
		-動画ファイル出力開始／停止は'w'キーで操作する
		-CSVファイル出力開始／停止は't'キーで操作する

## ９）射形動画を学習済AI(GRU)モデルで射法八節姿勢解析し、動画に表示・再生する

		>yoloap -gru -  -sample 1.10  
		
		-学習済モデル(GRU)ファイルに既定のファイル('-')を指定
		-データサンプリングに1.10を指定
		-'-case'パラメータを指定することで再生動画ファイル、解析データファイル出力を指定できる
		-'-level'パラメータを指定することで、プログラムロジック解析と併用して射法八節姿勢解析を行う（最も解析精度が高い）

## １０）解析済データ（ケース名）を確認して、解析データをデータベースに登録する（CSVファイルをDBにインポートする）

		>chart -list_case
		>chart -import yoshimoto_2m.0  
		または、  
		>kyudo -list case_name  
		>kyudo -import yoshimoto_2m.0 

		-'case_name'は名前リストのみ表示。'case'指定で詳細リストを表示。

## １１）解析結果（骨格ポイントデータ:tracking_data）をプロットする

		>chart -case yoshimoto_2m.0 -key right_wrist  
		
		-右手首のフレーム間移動量（ブロック高さで正規化）と、検出精度、節の移行状況グラフを上下にプロットする

		>chart -case yoshimoto_2m.0 -key 'right_wrist,left_wrist'  
		
		-右手首、左手首のフレーム間移動量（ブロック高さで正規化）のグラフを上下にプロットする  
		-キーポイントを４点指定したときは、グラフを上下、左右にプロットする  
		-次のコマンドでキーポイントの一覧を出力できる  
			>chart -list_key

## １２）解析結果（GRUモデル入力データ:kyudo_data）をプロットする

		>kyudo -case yoshimoto_2m.0 -input_key 80  
		
		-入力キー番号=80で定義されたモデル入力データを上下にプロットする
		-次のコマンドで定義済の入力キーの情報一覧を出力できる
		   >kyudo -list key

## １３）解析結果（射形評価データ:eval_data）をコンソール出力する

		>kyudo -eval 'yoshimoto_2m.9'  
		
		-打ちお越し、引き分け、会、残心における矢、腕の角度、保持時間、フレームNOなどをコンソールに出力する。
		-'*'指定で登録済のすべて、カンマ(,)区切りの複数指定で比較表示できる
		-5)のコマンドを使用して、当該データ検出時の姿勢を画像で表示、比較できる

		
## １４）GRUモデルの学習を実行する

		>kyudo -tarin yoshimoto_2m.0 -model kyudo2_80_modelse_8-96-3.pt 

		-'-train'オプションに'list'を指定したとき、既定の実行環境変数「登録済ケースリスト」を学習サンプルデータとする
		-「登録済ケースリスト」がスペース区切りの場合、各々を１サンプルデータとして扱い順次、追加学習を行う
		-「登録済ケースリスト」がカンマ区切りの場合、すべてを連結して１サンプルデータとして学習を行う
		-学習時のハイパーパラメータは既定の実行環境変数を参照する
		-指定モデルのファイルが存在しないときは新規の学習、存在するときは追加の学習を実行する 

## １５）学習済GRUモデルの予測結果を検証する		

		>kyudo -pridict yoshimoto_2m.0 -model kyudo2_80_modelse_8-96-3.pt 

		-'-predict'オプションで指定したケースの解析データをサンプルデータとして、指定モデルで予測した結果と比較する 

## １６）データベース登録済のケース（解析データ情報）を削除、リネームする

		>kyudo -rename yoshimoto_2m.0 -to yoshimoto_2m.0_new  
		>kyudo -delete yoshimoto_2m.0_new
